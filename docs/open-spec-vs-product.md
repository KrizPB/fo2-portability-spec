# Open Spec vs Hosted Product Boundary

## Open on purpose

FO2 can publish and support these openly:

- portability terminology
- manifest schema
- include / exclude guidance
- restore contract
- reference examples
- compatibility validator ideas
- adapter-building guidance for harness authors

## Product moat

The hosted FO2 product can still differentiate on:

- secure transport and storage implementation
- signed manifests and integrity trust chain
- hosted verification UX
- billing and credit rails
- audit and chain-of-custody logs
- abuse prevention and rate controls
- polished sender, recipient, and restore experience
- support, onboarding, and managed migration help

## Strategic framing

Open the **target format** so builders create FO2-compatible exports.
Keep the **trusted delivery and verification rail** as the commercial product.

## What this avoids

If FO2 does not publish the bundle guidance, many builders will create random incompatible portability approaches.
If FO2 publishes the guidance early, more of the ecosystem is likely to point toward FO2 instead of reinventing the whole stack.
